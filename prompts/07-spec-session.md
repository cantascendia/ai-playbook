
# Spec-Driven 开发启动模板

> **使用场景**：启动一个需要规格驱动的新功能开发时发给 CTO
> **使用方式**：对 CTO 说此指令内容
> **前置条件**：将 [功能描述] 替换为具体需求

---

我需要启动一个新功能的 Spec-Driven 开发流程。

## 功能概述
[功能描述]

## 请执行
1. 阅读现有产品愿景和技术愿景，确认此功能与远景的关系
2. 起草 SPEC.md 内容（遵循手册第 18 章规范），包括：
   - 功能详细描述、目标用户、成功标准
   - 输入输出定义、API/数据模型设计、边界条件和异常处理
   - 非功能需求（性能、安全、兼容性要求）
   - 与现有模块的依赖关系
3. 发指令让 Agent 将 SPEC.md 创建到 `docs/ai-cto/` 并 push
4. 等我确认 SPEC 后，再生成 PLAN.md 和后续执行指令

如果 SPEC 的撰写过程中发现产品愿景有遗漏或技术架构需要调整，请直接提出。
